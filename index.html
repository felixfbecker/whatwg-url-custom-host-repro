URL API Inconsistencies

Browsers behave differently with parsing out the host in the URL API if the protocol is a custom scheme, e.g. ssh://.

Input:

Browser url.protocol url.origin url.hostname url.pathname
Your browser
Chrome (69.0.3497.100) "ssh:" "ssh://" "" "//github.com/test/repo.git"
Firefox (63.0b14) "ssh:" "null" "" "//github.com/test/repo.git"
Edge (42.17134.1.0, EdgeHTML 17.17134) "ssh:" "ssh://" "github.com" "/test/repo.git"
Node (10.10.0) "ssh:" "null" "github.com" "/test/repo.git"
whatwg-url (7.0.0) "ssh:" "null" "github.com" "/test/repo.git"
